| Field     | Value                                                                            |
| --------- | -------------------------------------------------------------------------------- |
| Status    | Proposed                                                                         |
| Date      | 2026-05-02                                                                       |
| Scope     | `platform-ui-app/src/lib/fixtures/` + `sim/scenarios/silent-eye-20260502/`       |

---

## 1. Decision

We ship one battlegroup-scale scenario, `OP SILENT EYE`, as the canonical demo.

The data layer splits cleanly into two halves:

- **Pre-baked world state** in `sim/scenarios/silent-eye-20260502/silent-eye.events.jsonl` — entity spawns, sensor events, reports, unit telemetry. ~95 objects. Deterministic, reviewable, validated.
- **Runtime intelligence** generated by `platform-control-plane/` AI services as the world events arrive — fusion outputs, recommendations with cited evidence chains, copilot Q&A. Live LLM calls, not pre-baked strings.

A new `sim/replayer/` Go service paces the JSONL into the control-plane Ingest API at scaled wall-clock time.
The UI reads the live snapshot from the control-plane Read API and posts approvals back as TaskingOrders.

This split is the architectural payoff: the *sim* knows nothing about AI, the *control plane* is where intelligence lives, the *UI* is a thin viewer + action submitter.
Same UI works against pre-baked replay, PX4 SITL, or live field deployment — only the layer below ingest changes.

## 2. Why now

The current fixtures hold ~17 objects, which only proves the column layout renders.
Hand-authoring more rows piecemeal drifts the world inconsistently — orphan evidence refs, impossible kinematics, events without prequels — and reviewers spot the cracks.
This ADR locks the spec at the resolution needed for the data layer to be built without further design questions.

**Wire-format gotchas the replayer must honor** (from CP ADR 0003):

- `_id` derivation is server-side: producer omits `_id`, supplies `_source_ref`, server computes UUIDv5(`_source` + `_source_ref`) — deterministic across replay runs.
- The string IDs in this ADR (e.g., `ent_red_orlan_01`) become `_source_ref` values, not `_id`. The replayer must compute the same UUIDv5 client-side when populating `evidence_refs[]` so cross-references resolve.
- `_ingested_at` and `_version` are server-stamped — never sent.
- Server returns 200 on partial success with per-row errors; replayer logs and continues.
- 1000-row / 10 MB batch limit; replayer batches accordingly.

**Affiliation field**: TS-only on the UI side, not present in Go struct or ClickHouse. Replayer must not send it; UI computes from `_subtype` + `threat_level` + `_source` heuristics.
We deferr a CP-side `affiliation` column until ADR 0003 (runtime generator) needs it.

**No fusion / rec / copilot today**: these are greenfield in `internal/fusion/`, `internal/recengine/`, `internal/copilot/`, `internal/wsfeed/`. Tracked in §18 DoD.

## 14. Validator

`sim/scenarios/silent-eye-20260502/validate.py` walks the JSONL graph and fails on any of:

1. An `evidence_refs` ID not present somewhere earlier in the trace.
2. An `entity_id` referenced before any Entity record spawns it.
3. A Recommendation that references evidence chronologically *after* itself.
4. A Mission running past its `MissionObjective.deadline`.
5. A doctrinal-rule violation:
   - `casevac_request` without a prior `small_arms_contact` or `artillery_impact` within 60 min in the same area.
   - `unit_destroyed` without either a prior `*_strike` event in the same area or a doctrinal source (mine, ammo cook-off).
   - `breach_attempt` without prior `smoke_screen` or `artillery_impact`.
   - `withdrawal` without either prior `unit_destroyed` count ≥ 2 on the withdrawing side or `comms_outage` on its command element.
6. An arc-outcome violation: `red_breach_repelled` requires net RED `unit_destroyed` ≥ 4 AND no `withdrawal` of BLUE main element AND DeepState delta = 0 in AO.
7. A `gating: 'forbid-llm'` recommendation with `decided_by` that looks LLM-shaped (sanity gate on the OAG contract).

The validator runs before commit; the JSONL is treated as a build artifact that fails CI on any rule violation.

### 16.1 DeepStateMap (primary OSINT anchor)

The cyterat/deepstate-map-data GitHub repo provides 658 daily MultiPolygon GeoJSONs covering 2024-07-08 → 2026-05-02 (Russian-occupied territory of Ukraine, updated daily at 03:00Z).
Local copy lives at `~/Downloads/deepstate-map-data-main/`.

Two days are committed to the repo:

| File | Role |
| ---- | ---- |
| `src/lib/fixtures/deepstate-occupied-20260501.json` | Day-before frontline (validator input only) |
| `src/lib/fixtures/deepstate-occupied-20260502.json` | Demo-day frontline (rendered as map underlay + validator input) |

Three concrete uses for the SILENT EYE scenario:

1. **Visual map underlay.** `col-map.tsx` renders the 2026-05-02 polygon as the threat-tinted hatched layer. Operator sees the *actual* front line on that date, not a fictional one.
2. **Outcome certification.** The validator (§14 rule 6) clips both polygons to the AO bounding box, computes the symmetric difference, and asserts area < 0.1 km². If the actual front shifted, the "breach repelled" claim cannot stand and the build fails.
3. **Unit-placement sanity.** Validator computes each Unit/Entity position relative to the polygon: BLUE units must be outside RED-controlled area; RED entities at reveal time must be inside or within ~3 km of the polygon edge (the "contested grey zone"). Hand-edits that accidentally place ALPHA-6 in occupied territory are caught.

The remaining 656 days of the archive are not used by SILENT EYE but enable future scenarios:

| Future use | What the archive enables |
| ---------- | ------------------------ |
| Scenario date selection | Scan all 658 snapshots for days with the desired delta pattern (e.g., zero-delta for "repelled" arcs, ≥ 0.5 km² shift for "breakthrough" arcs) |
| Adversary intent inference | Compute polygon trajectory over weeks; the dominant axis of advance becomes the natural attack vector for fictional scenarios |
| Historical context overlay | "AO has been contested for 920 days; RED has gained 3.7 km² over the last 30 days" — derived directly from the archive |
| Delta-to-event decomposition (ADR 0004) | Each daily delta becomes 5–20 synthetic events with intra-day timestamps |
